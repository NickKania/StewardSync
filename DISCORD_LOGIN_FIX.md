# Discord Login Timeout Fix - Summary

## Problem

When deployed to production (Vercel), the Discord login was timing out with the error:
```
Error: Timed out waiting for Convex mutation. Check Convex backend connection.
```

## Root Cause

The application was trying to connect to `http://127.0.0.1:3210` (localhost) instead of the actual Convex Cloud deployment URL. This happened because:

1. `runtime-config.js` was empty: `window.__STEWARDSYNC_CONFIG__ = window.__STEWARDSYNC_CONFIG__ || {};`
2. The `ConvexService` used `appRuntimeConfig.convexUrl`, which fell back to `environment.convexUrl`
3. The production `environment.prod.ts` had a placeholder URL instead of the actual deployment URL

## Solution

The fix implements a proper environment variable-based configuration system that:

1. **Removes all hardcoded values** from environment files
2. **Generates configuration at build time** from environment variables
3. **Works with Vercel's deployment system**

## Changes Made

### 1. Environment Files

All environment files now have empty/default values and are generated at build time:

- `src/environments/environment.ts` - Development fallback (empty values)
- `src/environments/environment.prod.ts` - Production (generated by Vercel build)
- `src/environments/environment.staging.ts` - Staging (generated by staging build)
- `src/environments/environment.local.ts` - Local development (has localhost defaults)

### 2. Build Scripts

Created/Updated build scripts to generate environment files from environment variables:

- `scripts/vercel-build.sh` - **NEW** - Vercel production build script
- `scripts/build-prod.sh` - **NEW** - Local production build script
- `scripts/generate-env.sh` - Updated - Staging build script
- `scripts/generate-runtime-config-local.sh` - **NEW** - Local dev runtime config
- `scripts/build-with-env.sh` - **NEW** - Docker build with env variables

### 3. Vercel Configuration

Updated `vercel.json`:
- Changed build command to use `scripts/vercel-build.sh`

### 4. Runtime Configuration

Updated `src/app/core/config/runtime-config.ts`:
- Now properly falls back to `environment.convexUrl` when `runtime-config.js` is empty

### 5. Package.json

Updated build scripts:
- `start`, `start:local`, `start:staging` now generate `runtime-config.js` before starting

## How It Works

### Vercel Deployment

1. Vercel builds the application by running `scripts/vercel-build.sh`
2. The script generates `src/environments/environment.prod.ts` from environment variables
3. The script also generates `src/runtime-config.js` for browser-side configuration
4. Angular builds with the correct configuration
5. On browser load, `runtime-config.js` provides the Convex URL

### Local Development

1. Run `bun run start` or `bun run start:local`
2. Script generates `runtime-config.js` from `environment.local.ts`
3. Angular dev server starts with correct configuration

## Required Environment Variables

For Vercel deployment, set these in Vercel â†’ Settings â†’ Environment Variables:

| Variable | Description | Example |
|----------|-------------|---------|
| `PUBLIC_CONVEX_URL` | Your Convex deployment URL (REQUIRED) | `https://your-deployment.convex.cloud` |
| `PUBLIC_ENABLE_DEV_LOGIN` | Enable dev login bypass | `false` |
| `DISCORD_CLIENT_ID` | Discord OAuth client ID | `123456789012345678` |
| `DISCORD_CLIENT_SECRET` | Discord OAuth client secret | `your_secret_here` |

## Deployment Steps

### First Time Setup

1. Get your Convex deployment URL from [dashboard.convex.dev](https://dashboard.convex.dev)
2. Go to your Vercel project â†’ Settings â†’ Environment Variables
3. Add `PUBLIC_CONVEX_URL` with your Convex URL
4. Add other optional variables if needed
5. Redeploy your application

### Subsequent Deploys

Any changes to environment variables in Vercel will be picked up on the next deploy automatically.

## Testing

After deploying:

1. Open your Vercel deployment URL
2. Try to login with Discord
3. Should complete successfully without timeout

If still timing out:

1. Check browser console for errors
2. Verify `runtime-config.js` is loading correctly (Network tab)
3. Verify `PUBLIC_CONVEX_URL` is set in Vercel
4. Check Convex deployment is active at dashboard.convex.dev
